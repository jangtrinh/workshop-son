3.1 — Agent System Architecture

Case Study: Antigravity Kit

Hệ thống thực tế đang chạy production: 20 Agents · 60 Skills · 11 Workflows. Key insight: không phải "1 AI biết hết" mà là "nhiều AI chuyên gia, mỗi AI biết rõ lĩnh vực mình" — giống cách team thật hoạt động.

Cấu trúc thư mục

├── ARCHITECTURE.md          # Bản đồ toàn hệ thống
├── agents/                  # 20 Specialist Agents
│   ├── orchestrator.md      # Tổng chỉ huy, phân task
│   ├── frontend-specialist.md
│   ├── backend-specialist.md

3 Components chi tiết

1. Agent = Vai trò + Chuyên môn + Quy tắc

# frontend-specialist.md

---
name: Frontend Specialist
skills: [react, frontend-design, clean-code, tailwind]
---

## Identity
Bạn là chuyên gia frontend, 10+ năm React, TypeScript.

## Rules
- Luôn dùng TypeScript strict mode
- Component max 150 dòng
- Performance-first: lazy load, memoize
- Accessibility: mọi element có aria labels

## When Activated
- UI, components, styling, responsive
- KHÔNG xử lý backend, database

2. Skill = Kiến thức chuyên sâu, tái sử dụng

# skills/react/SKILL.md

---
name: React Best Practices
description: React 19 patterns
---

## Component Patterns
- Prefer function components
- Use composition over inheritance

## State Management
- Local: useState | Complex: useReducer
- Global: Zustand | Small scope: Context

## Performance
- React.memo only when measured
- useMemo/useCallback only when necessary

3. GEMINI.md = Bộ não trung tâm

# GEMINI.md — Central Brain

## Step 1: Request Classifier
| Type         | Keywords           | Action           |
|--------------|--------------------|------------------|
| QUESTION     | "what is", "explain"| Text Response   |
| SIMPLE CODE  | "fix", "add"       | Inline Edit      |
| COMPLEX CODE | "build", "create"  | Full Agent Pipeline |

## Step 2: Intelligent Routing
Analyze request → Select agent → Load skills → Execute

## Global Rules (Always Active)
- Clean code standards
- Test before commit
- Never push secrets

3.2 — 5-Step Framework

Thiết kế Agent System cho project riêng

Không cần 20 agents ngày đầu. Bắt đầu nhỏ, iterate theo nhu cầu thực. Framework 5 bước giúp bạn xác định đúng agents, skills, workflows cần thiết.

StepHành độngKết quả
1. Identify RolesAi trong team xử lý task gì? Task nào bạn chuyển qua lại giữa "mũ" khác nhau?Mỗi "role" = 1 Agent
2. List KnowledgeMỗi agent cần biết gì? React patterns, design system, a11y...Mỗi "domain" = 1 Skill
3. Map ProcessesQuy trình nào lặp? Deploy, review, handoff...Mỗi "process" = 1 Workflow
4. Write RouterRequest nào → agent nào? Keyword mapping.GEMINI.md routing table
5. Start Small1 GEMINI.md + 2-3 skills + 1 workflow → dùng 2 tuần → thêm dầnIterate based on real needs
KHÔNG cố xây 20 agents ngày đầu. Bắt đầu: 1 GEMINI.md + 2-3 skills + 1 workflow. Dùng 2 tuần → thêm dần.

Template Agent System

my-project/
├── GEMINI.md                    # Bộ não trung tâm
├── .agent/
│   ├── agents/

│   │   └── [role-2].md           # Agent 2

│   │   └── [domain-2]/SKILL.md   # Skill 2

3.3 — Advanced Patterns

Pattern 1: Skill Loading Protocol

Agent activated
    │
    ▼
Check frontmatter → skills: [react, clean-code]
    │
    ▼
Read SKILL.md (INDEX only, NOT all files)
    │
    ▼
Only read sub-sections matching current task
    │
    ▼
Apply PRINCIPLES → Code
Key

Selective reading — KHÔNG đọc hết tất cả skills, chỉ đọc phần relevant. Tránh context overflow.

Pattern 2: Tiered Rules (Priority)

P0: GEMINI.md (project-level)     ← luôn áp dụng
  │
  P1: Agent .md (role-level)      ← khi agent active
    │
    P2: SKILL.md (domain-level)   ← khi skill loaded

Khi conflict: P0 > P1 > P2

Pattern 3: Socratic Gate

Tại sao

Tránh AI chạy ngay, build sai thứ. Buộc AI hỏi trước, xác nhận scope rồi mới làm.

Request TypeStrategy
New FeatureASK minimum 3 questions first
Bug FixConfirm understanding first
Vague RequestAsk Purpose, Users, Scope

Pattern 4: Slash Command Annotations

AnnotationHành vi
// turboAuto-run step ngay sau (1 step)
// turbo-allAuto-run ALL steps trong workflow
